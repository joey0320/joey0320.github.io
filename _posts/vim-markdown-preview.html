Container Part 2 - Container Internals

1. Overview

- client : docker user, sends requests using cli - daemon (docker engine) : responsible for image management & build, authentication, security, core networking(constructs default bridge, docker0 etc), orchestration, uses the REST API to get requests from the client - containerd : controls the container life cycle - runc : cli that wraps around libcontainer, interacts with the kernel to construct namespaces, cgroups etc - shim : detaches runc from containerd(????)
